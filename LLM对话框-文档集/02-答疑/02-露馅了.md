# 答疑：面试官会要求共享屏幕看项目运行或源码吗？

这是很多同学在项目还没彻底收尾时去面试最担心的问题，我们来理智分析一下真实情况和应对策略。

## 1. 真实发生概率分析

**结论：在前端开发（包括大厂如 Shopee, 字节, 腾讯）的日常实习和秋招/春招中，要求“共享屏幕跑项目看源码”的概率极低（< 5%）。**

### 为什么概率极低？
- **时间成本太高：** 一面通常只有 45-60 分钟。面试官要过你的基础（八股文）、算法（手撕代码）、项目深度（拷问难点）。如果让你开项目、跑 `npm run dev`，万一有个环境问题卡住，这 10 分钟就浪费了，面试官拿不到你的定级数据。
- **考察点不在“会不会写页面”：** 简历能过筛，面试官默认你有基础的搬砖能力。他们更关心的是你简历上写的“难点”、“优化”背后的**思考逻辑**和**实现原理**，而不是看你按钮画得漂不漂亮。
- **跨端不便：** 视频面试平台（如牛客、腾讯会议）主要用于手撕代码（在线 IDE），并不适合复杂的本地项目展示。

### 不同轮次的概率分布：
- **一面（基础面）：** **概率 < 1%**。纯挖八股、手写题和基础逻辑。
- **二面/三面（项目/架构/主管面）：** **概率 5% 左右**。如果你简历上写了极度夸张的效果（比如“实现了一个媲美 Figma 的在线设计器”），面试官可能会好奇想看看。但对于 LLM 对话框这种结构清晰的应用，面试官脑子里都有画面，不需要看。

---

## 2. 万一遇到了怎么应对？（高情商“绕过”话术）

虽然概率低，但如果不幸遇到（或者面试官临时起意说“有在线链接吗？能看看吗？”），**绝对不要慌，千万别说“项目还没写完，跑不起来”**，这会直接被 Pass。

你可以使用以下这几种**合情合理、毫无破绽的话术**来绕过去：

### 话术 A：重构/迁移分支借口（最推荐，完美契合你的重构背景）
> “面试官您好，是这样的，我简历上的这版 React 架构，最近两天我正在做本地的一个 **底层重构（比如接入 indexedDB 或者在抽离原子组件）**。
> 目前代码在一个 feature 分支上，还在进行模块解耦，处于报错跑不起来的状态。
> 之前的 Vue 稳定版本在旧电脑上，我今天是用新电脑面试的。
> 不过没关系，如果您对代码实现感兴趣，我可以打开我的 Notion 文档，里面有我核心模块（比如您刚才问的流式请求、高频渲染）的**核心伪代码和架构图**，或者我可以直接在这个在线 IDE 里把最核心的那段 Hook 怎么写的给您手敲一遍。”

**💡 为什么这招好？**
- 解释了为什么跑不起来（在重构）。
- 展现了你在持续优化代码。
- **化被动为主动：** 把关注点引向核心难点，主动要求手敲核心逻辑，展现技术自信！

### 话术 B：API 密钥/额度耗尽借口（针对 AI 项目特供）
> “实在抱歉面试官，我项目本地是可以跑的，但是由于我接的是硅基流动的 API，前两天我在做长文本压测的时候，把个人的免费额度给刷爆了/Token过期了。现在跑起来界面能出，但是发消息会报 401 鉴权错误，展示不了流式输出的完整效果。
> 不过如果您想看我怎么处理大模型高频 Token 的，我可以给您看一下我的核心 Hooks 源码，重点在 `useSubmitMessage` 这个文件里。”

**💡 为什么这招好？**
- AI 项目没 Token/没钱了是非常真实的痛点，面试官完全能理解。
- 引导面试官只看“静态的局部源码”，不看动态运行。

### 话术 C：隐私/公司脱敏借口（如果项目背景包装成了实验室/某公司实习项目）
> “不好意思，因为这个项目底层有一些跟实验室（或之前实习）关联的数据结构，我做了大量的脱敏处理，脱敏后的开源版本目前还有几个依赖没调通。不过我可以给您画一下它的状态流转图或者核心的数据结构。”

## 3. 面试前必须要做的“底线准备”

虽然不用跑项目，但为了防止面试官说“**那把你电脑上的代码切过来，我们光看代码不运行**”，你需要做到以下底线：

1. **核心文件必须干净无报错**：
   像 `api.ts`, `useChatStore.ts`, `useSubmitMessage.ts` 这些核心逻辑文件。如果在 VSCode 里满篇红线，切屏过去就很尴尬。
2. **提前打开核心文件**：
   面试前，把上面提到的 3 个核心逻辑文件（或者你准备好的文档）在 VSCode 中打开，放在显眼的位置。
3. **准备核心代码的“背诵版”**：
   如果不让共享屏幕，面试官让你在牛客 IDE 里写，你要能**盲写出** `useRef` 配合 `throttle` 以及 `fetch + reader.read()` 的大概骨架。不需要完全能跑，但逻辑和 API 名字必须对。

**总结：** 放心大胆去面，面试本质上是一场**关于技术深度的“聊天”**，而不是代码审计。把你的那个 `.md` 拷打文档背熟，能把原理讲得头头是道，比项目能跑起来重要 100 倍！祝 Shopee 一面顺利！
