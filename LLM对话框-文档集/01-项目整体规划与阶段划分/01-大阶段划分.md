为了确保项目能够稳健、有序地从 Vue 迁移到 React，我们将整个重构过程拆分为 **五个核心阶段**。每个阶段都有明确的目标和产出，确保我们始终处于可控状态。

---

### 📅 第一阶段：基础设施搭建 (Foundation Phase)
**目标**：搭建一个配置完备、规范统一的空项目，为后续开发打好地基。

1.  **项目初始化**
    *   使用 `Vite` 创建 `React + TypeScript` 项目。
    *   配置 `tsconfig.json` (路径别名 `@/*`) 和 `vite.config.ts`。
2.  **样式系统配置**
    *   初始化 `Tailwind CSS`。
    *   初始化 `Shadcn/ui` 并配置 `cn` (classname) 工具函数。
    *   配置全局 CSS 变量（主题色、字体等）。
3.  **核心库安装**
    *   安装 `Zustand` (状态), `React Router` (路由), `Axios` (请求), `Lucide React` (图标)。
    *   配置 `ESLint` + `Prettier` 规则。
4.  **目录结构创建**
    *   按照技术文档建立 `src/components`, `src/store`, `src/lib`, `src/views` 等文件夹。

---

### 🧱 第二阶段：状态管理与逻辑核心 (Core Logic Phase)
**目标**：不碰 UI，先把“大脑”移植过来。确保数据流和业务逻辑在 React 中跑通。

1.  **迁移 Settings Store (Zustand)**
    *   将 Vue 的 `setting.js` 转换为 `useSettingsStore.ts`。
    *   实现 API Key、模型参数的持久化存储 (`persist`)。
2.  **迁移 Chat Store (Zustand)**
    *   将 Vue 的 `chat.js` 转换为 `useChatStore.ts`。
    *   实现会话列表管理 (CRUD)、当前会话切换、消息追加逻辑。
3.  **API 服务层封装**
    *   重写 `api.js` 为 `lib/api.ts`。
    *   **难点攻克**：实现流式请求 (`Streaming`) 的处理逻辑，编写一个通用的 `streamCompletion` 函数，使其能对接 Zustand 更新消息内容。
4.  **单元测试/逻辑验证**
    *   创建一个临时测试组件，手动调用 Store 方法，验证数据是否能正确保存到 LocalStorage。

---

### 🎨 第三阶段：UI 组件与布局 (UI Component Phase)
**目标**：使用 Shadcn/ui 搭建界面骨架，还原视觉效果。

1.  **基础组件引入**
    *   使用 Shadcn CLI 安装需要的组件：`Button`, `Input`, `Dialog`, `Sheet` (抽屉), `DropdownMenu`, `Textarea`。
2.  **布局组件开发**
    *   `Layout.tsx`：实现侧边栏 (Sidebar) + 主内容区 (Main) 的响应式布局。
    *   `Sidebar`：实现历史会话列表的渲染、新建会话按钮。
    *   `Header`：实现顶部导航栏、设置按钮、移动端菜单触发器。
3.  **核心业务组件**
    *   `ChatInput`：重写输入框组件，支持 Textarea 自适应高度、发送快捷键监听。
    *   `MessageBubble`：实现单条消息的气泡样式，区分 User/Assistant 样式。
    *   `MarkdownRenderer`：集成 `react-markdown`，实现消息内容的富文本渲染和代码高亮。

---

### 🧩 第四阶段：功能集成与页面组装 (Integration Phase)
**目标**：将 UI 与 逻辑 (Store) 连接起来，实现完整交互。

1.  **首页与聊天页路由**
    *   配置 React Router，定义 `/` (首页/空状态) 和 `/chat/:id` (对话页)。
2.  **对话核心流程打通**
    *   **发送**：ChatInput 触发 -> 调用 API -> 开启流式读取。
    *   **接收**：流式回调 -> 实时更新 Store -> MessageBubble 实时渲染打字机效果。
3.  **设置功能实现**
    *   开发 `SettingsDialog` (或 Drawer)，使用 `react-hook-form` 绑定设置数据，关联 `useSettingsStore`。
4.  **辅助功能迁移**
    *   会话标题编辑、删除会话功能。
    *   消息重新生成 (Regenerate) 功能。
    *   自动滚动到底部 (Auto-scroll) 的 Hooks 实现。

---

### 🧹 第五阶段：优化与打磨 (Polishing Phase)
**目标**：修复 Bug，优化性能，提升用户体验。

1.  **样式微调**
    *   适配暗黑模式 (Dark Mode)。
    *   优化移动端适配 (Responsive Design)。
2.  **性能优化**
    *   使用 `React.memo` 优化长列表渲染（避免输入时整个聊天记录重绘）。
    *   检查 Zustand 的 Selector 使用，避免不必要的重渲染。
3.  **清理与交付**
    *   删除无用的临时文件。
    *   编写 README.md。
    *   最终测试所有功能路径。

---

**建议**：我们按照**阶段一 -> 阶段二 -> ...** 的顺序稳步推进。每完成一个阶段，都是一个可运行的里程碑。
