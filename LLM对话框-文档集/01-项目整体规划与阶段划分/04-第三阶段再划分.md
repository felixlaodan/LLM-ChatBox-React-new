### 🎨 第三阶段：UI 组件与布局拆解

#### 3.1 小阶段：基础 UI 原子组件准备
**做什么**：
不需要自己手写基础组件，我们要使用 Shadcn/ui 的 CLI 工具，把聊天应用需要用到的所有基础原子组件下载到本地。
**具体任务**：
- 运行 Shadcn 命令安装所需组件：`Button` (按钮), `Input` (单行输入框), `Textarea` (多行输入框), `Dialog` (弹窗，用于设置), `Sheet` (抽屉，用于移动端侧边栏), `Avatar` (头像), `ScrollArea` (自定义滚动条), `DropdownMenu` (下拉菜单), `Tooltip` (提示框)。
**验收标准**：
- `src/components/ui/` 目录下成功生成了上述所有组件的 `.tsx` 文件。
- 在 `App.tsx` 中随意引入一个 `Button`，运行后页面上能正常显示且带有默认的 Hover 样式，无任何报错。

#### 3.2 小阶段：全局响应式布局骨架 (App Layout)
**做什么**：
搭建整个应用的最外层结构，通常是一个经典的“左侧边栏 + 右侧主内容区”的布局。
**具体任务**：
- 创建 `src/components/layout/AppLayout.tsx`。
- 划分布局：左侧是固定宽度的侧边栏（例如 `w-64`），右侧是占据剩余空间的主区域（`flex-1`）。
- 响应式处理：在移动端（`< md`），隐藏左侧边栏，右侧主区域占满全屏。
- 限制全局高度：确保整个容器是 `h-screen` 和 `overflow-hidden`，防止整个浏览器页面出现难看的滚动条。
**验收标准**：
- 电脑端查看，屏幕被清晰地分为左右两块。
- 手机端查看（或缩小浏览器窗口），左侧边栏消失，右侧区域占满全屏。页面整体无法上下滚动。

#### 3.3 小阶段：侧边栏与历史会话列表 (Sidebar & History)
**做什么**：
在刚才画好的左侧区域，实现对历史对话的管理。
**具体任务**：
- 创建 `src/components/layout/Sidebar.tsx`。
- 顶部放置一个醒目的 “新建对话 (New Chat)” 按钮。
- 读取 `useChatStore` 中的 `conversations` 列表，使用 `Array.map` 渲染出会话列表。
- 高亮显示当前正在选中的会话（依据 `currentConversationId`）。
- 给会话列表加上 `ScrollArea`，确保对话很多时只有列表区域滚动。
**验收标准**：
- 侧边栏能正确显示 Store 中的所有对话标题。
- 点击不同的对话，能在 Store 中触发 `switchConversation`（可以通过 React DevTools 观察状态变化，或者在控制台打印）。
- 点击“新建对话”，列表头部会立刻出现一个新的对话项，并且变为选中状态。

#### 3.4 小阶段：主界面头部与设置弹窗 (Header & Settings)
**做什么**：
搭建右侧主区域的顶部导航条，并实现配置 API Key 的设置面板。
**具体任务**：
- 创建 `src/components/layout/ChatHeader.tsx`。左侧显示当前对话的标题，移动端时左侧额外增加一个汉堡菜单按钮（点击通过 `Sheet` 呼出侧边栏）。
- 创建 `src/components/business/SettingsDialog.tsx`。
- 在头部右侧放一个“设置”图标，点击呼出设置弹窗。
- 弹窗内使用 `react-hook-form` 渲染表单，包含：API Key、模型选择（下拉框）、流式开关、Temperature 等。表单的初始值和提交操作需要绑定到 `useSettingsStore`。
**验收标准**：
- 头部能正确显示当前对话标题。移动端下能通过汉堡按钮拉出左侧栏。
- 打开设置弹窗，修改 API Key 或模型后点击保存，刷新页面后打开设置，修改的值依然存在（证明 Zustand Persist 生效且绑定正确）。

#### 3.5 小阶段：消息气泡与对话列表 (Message List & Bubbles)
**做什么**：
开发聊天区域的核心部分——用于展示用户和 AI 消息的对话框。
**具体任务**：
- 创建 `src/components/business/MessageBubble.tsx`，负责单条消息的渲染。需要根据角色 (`role === 'user'` 或 `role === 'assistant'`) 给予不同的样式（如：用户在右侧/蓝色背景，AI 在左侧/灰色背景，配上对应的头像 Avatar）。
- 创建 `src/components/business/MessageList.tsx`，读取 `currentMessages` 渲染消息列表。
- **关键交互**：在这个组件中编写 Auto-scroll（自动滚动到底部）的逻辑。当 `messages` 数组长度变化，或者流式生成时消息内容变化，自动将滚动条推到最底部。
**验收标准**：
- 在 Store 中手动塞入几条假消息，页面能正确区分左右展示，UI 美观，没有撑破容器。
- 无论假消息有多少，进入页面时或新增消息时，视图会自动定位到最后一条消息。

#### 3.6 小阶段：Markdown 与代码高亮渲染集成
**做什么**：
大模型的回复通常是 Markdown 格式的，我们需要把这些生硬的标记语言变成漂亮的富文本和带语法高亮的代码块。
**具体任务**：
- 创建 `src/components/business/MarkdownRenderer.tsx`。
- 引入 `react-markdown` 渲染基础 Markdown，引入 `remark-gfm` 支持表格，引入 `rehype-highlight` 支持代码高亮。
- 深度定制代码块的渲染，最好能在代码块右上角加一个“复制 (Copy)”按钮。
- 将 `MessageBubble` 中 AI 的回复内容替换为用 `MarkdownRenderer` 组件渲染。
**验收标准**：
- 往 Store 里塞一条包含 `**加粗**`、`[链接](#)`、表格以及 ```javascript 完整代码块 ``` 的假消息。
- 页面能完美渲染出加粗、表格样式，代码块有语法高亮背景色，且点击复制按钮能成功将代码复制到剪贴板。

#### 3.7 小阶段：底部聊天输入区 (Chat Input)
**做什么**：
实现用户输入问题、发送消息的底部输入框。
**具体任务**：
- 创建 `src/components/business/ChatInput.tsx`。
- 使用 Shadcn 的 `Textarea`，并实现**自动伸缩高度**功能（随着用户打字换行，输入框长高，但达到最大高度后内部滚动）。
- 实现快捷键支持：按 `Enter` 发送消息，按 `Shift + Enter` 换行。
- 添加一个“发送”按钮，在 `isLoading`（也就是大模型正在回复时）时，按钮变为“停止生成”或呈禁用/Loading 状态。
**验收标准**：
- 输入框固定在主区域最底部。
- 输入长文本时输入框会优雅地长高。
- 按回车能触发发送事件（暂时用 `console.log` 测试即可），并在发送后清空输入框内容并将高度还原。
