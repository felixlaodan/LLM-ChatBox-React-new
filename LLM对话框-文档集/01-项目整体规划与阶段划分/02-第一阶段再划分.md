好的，我们把 **第一阶段：基础设施搭建 (Foundation Phase)** 详细展开。这个阶段是项目的“各种配置文件”和“脚手架”确立的过程，虽然写不了多少业务代码，但对后续开发的顺畅度至关重要。

我们将第一阶段细分为 **3 个小阶段 (Milestones)**：

---

### 🟢 阶段 1.1：项目初始化与环境配置
**目标**：获得一个干净的、可运行的 React + TS 项目，并配好路径别名。

*   **步骤**：
    1.  **创建项目**：使用 `pnpm create vite` 生成项目骨架。
    2.  **清理代码**：删除 Vite 默认生成的 `App.css`、`index.css` 内容，删除 demo 代码（计数器等），保留最简 `App.tsx`。
    3.  **配置 TS**：在 `tsconfig.json` 和 `tsconfig.app.json` 中配置 `baseUrl` 和 `paths`，启用 `@` 指向 `src` 目录。
    4.  **配置 Vite**：在 `vite.config.ts` 中同步配置 alias 解析。
    5.  **安装 Node 类型**：安装 `@types/node` 以支持 Node.js 模块（如 `path`）在配置文件中的使用。

*   **🏆 验收标准**：
    *   执行 `pnpm dev` 无报错。
    *   在代码中 `import x from '@/components/x'` 不报错，且能正确跳转。
    *   浏览器看到一个空白页面或 "Hello World"。

---

### 🔵 阶段 1.2：样式系统与 UI 库接入
**目标**：接入 Tailwind CSS 和 Shadcn/ui，确立设计规范。

*   **步骤**：
    1.  **安装 Tailwind**：安装 `tailwindcss`, `postcss`, `autoprefixer`，生成配置文件。
    2.  **配置 Tailwind**：在 `tailwind.config.js` 中配置 content 扫描路径。
    3.  **引入全局样式**：在 `src/index.css` 中引入 `@tailwind` 指令。
    4.  **初始化 Shadcn**：运行 `shadcn-ui init`，配置 `components.json`。
    5.  **工具函数检查**：确认 `src/lib/utils.ts` (或类似路径) 已生成，里面包含 `cn` (classname 合并) 函数。
    6.  **测试组件**：试着安装一个 Button 组件 (`shadcn-ui add button`) 并在 App 中渲染。

*   **🏆 验收标准**：
    *   页面上能看到一个 Shadcn 风格的 Button。
    *   使用 Tailwind 类名 (如 `text-red-500`) 能立即生效。
    *   项目目录下出现了 `components/ui` 文件夹。

---

### 🟣 阶段 1.3：核心依赖安装与目录规范
**目标**：装好所有“全家桶”依赖，并建立符合架构文档的文件夹结构。

*   **步骤**：
    1.  **批量安装依赖**：
        *   **路由**：`react-router-dom`
        *   **状态**：`zustand`
        *   **请求**：`axios`
        *   **图标**：`lucide-react`
        *   **Markdown**：`react-markdown`, `rehype-highlight`
        *   **表单**：`react-hook-form`, `zod`
    2.  **创建目录结构**：
        手动创建以下空文件夹（保持 `.gitkeep` 或随意放个空文件）：
        *   `src/components/business` (业务组件)
        *   `src/components/layout` (布局组件)
        *   `src/store` (状态管理)
        *   `src/hooks` (自定义 Hooks)
        *   `src/views` (页面)
        *   `src/types` (类型定义)
        *   `src/assets` (静态资源)

*   **🏆 验收标准**：
    *   `package.json` 中包含上述所有依赖。
    *   `src` 目录结构清晰，符合架构文档规划。

---

**最终呈现结果 (Deliverable)**：
完成这三个小阶段后，你将拥有一个**“已装好引擎、内饰空空如也的赛车底盘”**。
*   你可以直接开始写 Store 和 组件，而不需要再停下来配置环境。
*   所有的 import 路径都是干净的（`@/...`）。
*   所有的 UI 基础样式（字体、颜色变量）都已就绪。
